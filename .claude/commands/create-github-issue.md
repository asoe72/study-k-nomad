# Create GitHub Issue

주어진 내용을 바탕으로 GitHub 이슈를 생성합니다.

## 생성할 이슈 내용

$ARGUMENTS

## 작업 절차

### Step 1 — 이슈 내용 분석

$ARGUMENTS를 분석하여 아래 항목을 파악합니다:

- **이슈 유형**: bug(버그), enhancement(기능 추가), documentation(문서), question(질문) 등
- **핵심 제목**: 이슈를 한 줄로 요약한 명확한 제목
- **상세 설명**: 이슈의 배경, 현상, 기대 동작, 재현 방법 등
- **레이블**: 이슈 유형에 맞는 적절한 GitHub 레이블
- **우선순위**: 긴급도 파악 (언급된 경우)

### Step 2 — 이슈 본문 작성

이슈 유형에 따라 적절한 마크다운 본문을 작성합니다.

**버그 리포트인 경우:**
```
## 버그 설명
[현상 설명]

## 재현 방법
1. ...
2. ...

## 기대 동작
[기대하는 동작]

## 실제 동작
[실제로 발생하는 동작]

## 환경
- OS:
- 버전:
```

**기능 요청인 경우:**
```
## 기능 개요
[기능 설명]

## 동기 및 배경
[이 기능이 필요한 이유]

## 제안하는 해결 방법
[구체적인 구현 방향]

## 완료 기준
- [ ] ...
- [ ] ...
```

**기타 이슈인 경우:**
```
## 개요
[이슈 설명]

## 상세 내용
[구체적인 내용]

## 기대 결과
[해결 후 기대하는 상태]
```

### Step 3 — gh CLI로 이슈 생성

아래 명령어로 GitHub 이슈를 생성합니다:

```bash
gh issue create \
  --title "[이슈 제목]" \
  --body "[작성한 본문]" \
  --label "[레이블]"
```

- 레이블이 저장소에 없을 수 있으므로, `--label` 옵션은 레이블이 확인된 경우에만 사용합니다.
- 먼저 `gh label list`로 사용 가능한 레이블을 확인하세요.

### Step 4 — 결과 보고

이슈 생성 후 다음을 사용자에게 보고합니다:

- 생성된 이슈 URL
- 이슈 번호
- 제목 및 적용된 레이블

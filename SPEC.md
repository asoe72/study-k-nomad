# k-nomad MVP 개선 실행 계획

## 현재 상태 요약

- Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui
- 페이지: `/`(홈), `/login`, `/register`
- 주요 데이터: `lib/data.ts` (7개 도시 mock 데이터)
- 현재 필터: `수도권`, `제주`, `바다`, `산·자연`, `저비용`
- 카드 구조: 5개 지표 바 + 생활비 + 날씨 + AQI + GOOD/BAD/리뷰 버튼

---

## Phase 목록

- [ ] Phase 1: 데이터 구조 재설계 및 Mock 데이터 업데이트
- [ ] Phase 2: 네비게이션 단순화
- [ ] Phase 3: 필터 시스템 재설계
- [ ] Phase 4: 카드 컴포넌트 재설계 + 좋아요/싫어요 기능
- [ ] Phase 5: 도시 리스트 섹션 정리

---

## Phase 실행 순서 의존 관계

```
Phase 1 (데이터 구조)
    ↓
Phase 2 (네비게이션)  ←  Phase 1과 독립적, 병렬 실행 가능
    ↓
Phase 3 (필터 시스템)  ← Phase 1 완료 필요
    ↓
Phase 4 (카드 재설계)  ← Phase 1 완료 필요
    ↓
Phase 5 (리스트 정렬)  ← Phase 4 완료 필요
```

> Phase 2는 Phase 1과 독립적이므로 동시에 진행 가능합니다.
> Phase 3, 4는 Phase 1 완료 후 병렬로 진행 가능합니다.
> Phase 5는 Phase 4의 좋아요 상태 구조 완료 후 진행합니다.

## Phase 3: 필터 시스템 재설계

### 오버뷰

`CitiesClient.tsx`의 기존 단일 태그 필터를 4개 카테고리 필터(예산 / 지역 / 환경 / 최고 계절)로 교체합니다. 각 카테고리는 독립적으로 선택 가능하며, 복수 카테고리 조건은 AND로 결합합니다. 필터 상태는 `useState`로 관리하고, Phase 1에서 추가한 데이터 필드를 기반으로 필터링 로직을 구현합니다.

### 수정/개선 사항

- [ ] `CitiesClient.tsx`에서 기존 `FILTERS` 배열 및 단일 필터 상태 제거
- [ ] 4개 카테고리 필터 상수 정의
  - 예산: `["전체", "100만원 이하", "100~200만원", "200만원 초과"]`
  - 지역: `["전체", "수도권", "경상도", "전라도", "강원도", "제주도", "충청도"]`
  - 환경: `["전체", "자연친화", "도심선호", "카페작업", "코워킹 필수"]`
  - 최고 계절: `["전체", "봄", "여름", "가을", "겨울"]`
- [ ] 4개 독립 필터 상태 추가 (`selectedBudget`, `selectedRegion`, `selectedEnvironment`, `selectedSeason`)
- [ ] 4개 카테고리별 필터 버튼 UI 구현 (카테고리 레이블 + 버튼 그룹)
- [ ] 필터 로직 구현 (각 카테고리 조건 AND 결합)
  - 예산 필터: `city.budget === selectedBudget` (전체 선택 시 무시)
  - 지역 필터: `city.region === selectedRegion` (전체 선택 시 무시)
  - 환경 필터: `city.environment.includes(selectedEnvironment)` (전체 선택 시 무시)
  - 계절 필터: `city.season.includes(selectedSeason)` (전체 선택 시 무시)
- [ ] 선택된 필터 버튼 활성화 스타일 적용 (기존 neon 색상 스킴 유지)
- [ ] 필터 결과가 없을 때 "해당하는 도시가 없습니다" 빈 상태 UI 표시

### 검증/확인 사항

- [ ] 4개 카테고리 필터 UI 정상 표시 확인
- [ ] 예산 필터 단독 동작 확인 (각 옵션 클릭 시 해당 도시만 노출)
- [ ] 지역 필터 단독 동작 확인
- [ ] 환경 필터 단독 동작 확인
- [ ] 최고 계절 필터 단독 동작 확인
- [ ] 2개 이상 카테고리 필터 조합 시 AND 조건 동작 확인
- [ ] "전체" 선택 시 해당 카테고리 필터 해제 확인
- [ ] 필터 결과 없음 상태 UI 확인
- [ ] 모바일 반응형 필터 UI 깨짐 없음 확인

## Phase 4: 카드 컴포넌트 재설계 + 좋아요/싫어요 기능

### 오버뷰

`CityCard.tsx`를 전면 재설계합니다. 기존의 5개 지표 바, 별점 평점, 자세히 보기 버튼을 제거하고, Phase 1에서 추가한 `budget`, `region`, `environment`, `season` 필드를 Key-Value 형태로 표시합니다. 좋아요/싫어요 버튼은 클릭 시 카운트가 변동하고 아이콘 색상이 변경되는 인터랙션을 구현합니다.

### 수정/개선 사항

- [ ] `CityCard.tsx`에서 `MetricBar` 컴포넌트 및 5개 지표 바 영역 전체 제거
- [ ] `CityCard.tsx`에서 별점/점수(`score`) 표시 영역 제거
- [ ] `CityCard.tsx`에서 날씨(`weather`), AQI 표시 영역 제거
- [ ] `CityCard.tsx`에서 "자세히 보기" 버튼 제거
- [ ] `CityCard.tsx`에서 기존 GOOD/BAD 버튼 제거 후 새 좋아요/싫어요 버튼으로 교체

- [ ] 좋아요/싫어요 상호 배타 로직 구현 (좋아요 누른 상태에서 싫어요 누르면 좋아요 해제)
